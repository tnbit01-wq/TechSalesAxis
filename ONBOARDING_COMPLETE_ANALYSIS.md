## DATABASE STORAGE SCHEMA

### Primary Table: `candidate_profiles`

```python
# Career Readiness Response Storage
current_employment_status: str         # Step 1: Employed/Unemployed/Student
job_search_mode: str                   # Step 2: exploring/passive/active
notice_period_days: int                # Step 3: 0/7/14/30/60/90/180
availability_date: datetime            # Calculated: today + notice_period_days
willing_to_relocate: bool              # Step 3: additional preference
career_readiness_timestamp: datetime    # When saved

# Career Readiness Metadata (JSONB)
career_readiness_metadata: {
  "exploration_trigger": str,          # Step 2: why searching
  "contract_preference": str,          # Step 4: fulltime/contract
  "visa_sponsorship_needed": bool,     # Step 4: needs visa?
  "salary_flexibility": float,         # Step 4: 0-1 scale

}

# Career Readiness Scoring
career_readiness_score: int            # 0-100 calculated score
employment_readiness_status: str       # Overall status
onboarding_step: str                   # Progress tracking: AWAITING_CAREER_READINESS → AWAITING_RESUME
```

## TEXT VISIBILITY ISSUE - ROOT CAUSE & FIX

### Issue
User reported: **"Response text is white and not visible"**

### Root Cause
Bot message text color was relying only on Tailwind class `text-slate-700` without inline style fallback. If Tailwind classes don't apply (build issue, cache, theme problem), text becomes invisible.

### Solution Applied
Added explicit inline styles to ensure visibility regardless of Tailwind parsing:

**File 1**: [apps/web/src/components/CareerReadinessFlow.tsx](apps/web/src/components/CareerReadinessFlow.tsx#L668)
```jsx
// Bot messages: dark gray text on light gray background
style={msg.sender === "bot" ? {
  color: "#374151",  // Dark gray
  backgroundColor: "#f8fafc"  // Light gray
} : {}}

// Button options: dark gray text on white background  
style={{
  color: "#1f2937",  // Dark gray
  backgroundColor: "#ffffff",  // White
  borderColor: "#e2e8f0"  // Light border
}}
```

**File 2**: [apps/web/src/app/onboarding/candidate/page.tsx](apps/web/src/app/onboarding/candidate/page.tsx#L1388)
- Same fixes applied to main onboarding page messages

### Verification
Run in browser console after navigating to onboarding:
```javascript

document.querySelector('[class*="bg-slate-50"]').style.color
// Should return: "rgb(55, 65, 81)" or similar dark gray
```
