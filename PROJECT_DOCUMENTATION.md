## Platform Overview

### What is TalentFlow?

**TalentFlow is a high-trust recruitment platform** that revolutionizes how companies and candidates connect by implementing a **dual-assessment model**:

- **Candidate Assessment:** AI-powered evaluation of skills, experience, and behavioral competencies
- **Recruiter Assessment:** Quality screening of hiring companies to ensure candidate trust

Both parties are "verified" before gaining full access, creating a premium, filtered marketplace rather than a traditional job board.

### Vision

To build a **verified talent marketplace** where:
- Candidates are genuinely skilled and assessed
- Recruiters are vetted for fair hiring practices
- Matching is deterministic and explainable
- Everyone operates within a trust-based system

### Target Users

| User Type | Purpose | Key Motivation |
|-----------|---------|-----------------|
| **Candidates** | Job search + career development | Find trustworthy companies, get fair assessment |
| **Recruiters** | Talent acquisition | Access verified talent pool, reduce hiring friction |
| **Companies** | Company profile management | Build trust through quality candidates |

## Core Business Concept

### The Two-Gate Model

TalentFlow operates on a **trust-based matching system** with two mandatory gates:

#### Gate 1: Candidate Assessment
```
Candidates CANNOT see jobs or apply until:
✓ Complete 6-step onboarding
✓ Pass AI-powered skill assessment (6-16 questions)
✓ Verify identity (Aadhaar/Passport)
✓ Accept terms
```

**Output:** Trust Score (0-100)  
**Locked Fields:** Experience band, core skills, psychometric profile

#### Gate 2: Recruiter Assessment
```
Recruiters CANNOT post jobs until:
✓ Verify company (CIN/GSTIN)
✓ Complete company bio (auto-generated by Gemini)
✓ Answer 5 strategic/ethical questions
✓ Achieve minimum Company Profile Score (CPS)
```

**Output:** Company Trust Score (0-100)  
**Benefit:** Determined hiring intent and fairness

### Why This Model?

| Problem | Solution |
|---------|----------|
| Fake candidates (resume inflation) | AI-powered assessment gates pipeline |
| Misleading companies (bait-and-switch jobs) | Recruiter assessment validates intent |
| Resume flooding (100s of low-quality apps) | Trust filter reduces noise 90%+ |
| Ghosting & miscommunication | Verified parties are incentivized to engage |
| Hiring bias | AI scoring removes subjective bias from initial screening |

## Candidate Experience

### Step-by-Step User Journey

#### Phase 1: Authentication
```
1. Visit TalentFlow
2. Click "Candidate Signup"
3. Enter email → OTP verification
4. Set password → Account created
5. Role selected: CANDIDATE
```

#### Phase 2: Onboarding (6-Step Gated)

**Step 1: Experience Band Selection**
- Choose: Fresher | Mid-Level | Senior | Leadership
- Determines assessment difficulty, job recommendations, skill expectations
- Cannot be changed after submission

**Step 2: Resume Intelligence**
- Option A: Upload PDF/DOC (parsed by Groq Llama in real-time)
- Option B: Build resume interactively (for freshers/new grads)
- Output: Structured resume data (roles, education, gaps, achievements)

**Step 3: Skill Enrichment**
- AI extracts skills from resume (highlighted)
- Add manual skills (verified via assessment later)
- Confirm expertise level per skill (Beginner/Intermediate/Advanced)
- Min. 3 skills required

**Step 4: Assessment Execution**
- **Duration:** 30-45 minutes
- **Questions:** Dynamically generated based on experience band
  - Fresher: 6 questions (lighter load)
  - Mid-Level: 10 questions
  - Senior: 14 questions
  - Leadership: 16 questions
- **Categories:** Resume challenges, skill case studies, behavioral, psychometric
- **Scoring:** Real-time by Gemini AI (0-6 per answer, normalized to 0-100 final score)
- **Result:** Trust Score displayed immediately
- **Anti-Cheat:** Tab-switches monitored (1 warning → 2nd attempt blocked)

**Step 5: Identity Verification**
- Upload: Aadhaar (with masking) OR Passport
- AWS RDS secure storage
- Not visible to recruiters but flags profile as "verified"

**Step 6: Terms Acceptance**
- Agree to: Community guidelines, assessment integrity, data privacy
- Accept notifications consent
- Profile goes ACTIVE

#### Phase 3: Dashboard Access
Once onboarding complete, candidates see:

**Dashboard (Home)**
- Profile completion % (aim for 100%)
- Trust Score badge
- Active status
- Recommended jobs (5 cards)
- Recent application status updates

**Jobs (Recommended)**

**Applications**

- Timeline view
- Status changes trigger notifications

**Community**

- Like, comment, share
- Follow companies for updates

**Profile**
- View resume (as recruiter sees it)
- Edit skills (can add new, confirm expertise)
- View trust score components
- Add/edit personal info (gender, employment status, referral source)
- Career goals (target role, long-term vision)

**Career Development**
- Skill recommendations: Skills to develop
- Career progression: Next-step roles
- Salary benchmarks: Role industry standards

**Messages**
- Chat threads (recruiter-initiated)
- Real-time messaging (WebSockets + Database Polling)
- One-to-one communication only
- Available after shortlist or interview invite

**Notifications**
- Signal Center: All alerts in one place
- Types: Application updates, messages, system alerts
- Channels: Email, in-app, mobile push

**Settings**
- Language & timezone

## Recruiter Experience

### Step-by-Step User Journey

#### Phase 1: Authentication
```
1. Visit TalentFlow
2. Click "Recruiter Signup"
3. Enter email → OTP verification
4. Set password → Account created
5. Role selected: RECRUITER
```

#### Phase 2: Company Setup (Assessment-Gated)

**Step 1: Company Identification**
- Enter: Company name, CIN (Unique Identification), GSTIN (Tax ID)
- Regex validation against GOI (India) standards
- Stores company in `companies` table

**Step 2: Auto-Generated Company Bio**
- TalentFlow's Gemini AI:
  - Scrapes company website
  - Synthesizes a professional bio
  - Shows draft to recruiter for edits
- **Output:** Company description (used in candidate job view)

**Step 3: Recruiter Assessment (5 Strategic Questions)**
- Questions sampled from 125-question bank
- **Categories & Sample Questions:**
  1. **Strategic Intent:** "Where will you expand hiring in the next 2 years?"
  2. **ICP (Ideal Candidate Profile):** "Describe the ideal candidate for backend roles."
  3. **Ethics & Fairness:** "How do you ensure unbiased hiring?"
  4. **Candidate Value Prop:** "What unique benefits do you offer candidates?"
  5. **Team Fit & Ownership:** "Who is accountable for candidate experience?"
- **Scoring:** Gemini AI grades 0-6 per question (same rubric as candidate assessment)
- **CPS (Company Profile Score):** Normalized 0-100
- **Logic:** Never decreases (MAX function; if Recruiter 2 scores lower, keep old score)

**Step 4: Dashboard Unlock**
- Option A: Complete assessment → Full dashboard access
- Option B: "Do It Later" → Basic features only (limited)

**Account Settings**

### Security: Row-Level Security (RLS) Policies

**Principle:** Users can only see their own data or data explicitly shared.

```sql
-- Candidates see only their own profile
candidates_select_policy:
  auth.uid() = candidate_profiles.user_id

-- Recruiters see candidates in their job's applications
jobs_candidates_policy:
  job_applications.job_id 
  IN (SELECT id FROM jobs WHERE company_id IN 
      (SELECT company_id FROM recruiter_profiles 
       WHERE user_id = auth.uid()))

-- Users see messages only if they're party to thread
messages_policy:
  chat_threads.recruiter_id = auth.uid() 
  OR chat_threads.candidate_id = auth.uid()

-- Candidates cannot see another candidate's assessment
assessment_isolation_policy:
  assessment_sessions.candidate_id = 
    (SELECT id FROM candidate_profiles WHERE user_id = auth.uid())
```

## AI Integration

### 1. Assessment Scoring (Google Gemini 1.5 Flash)

**Purpose:** Evaluate candidate answers to assessment questions objectively.

**Process:**
```
1. Candidate submits answer to question
2. Backend sends to Gemini:
   - Question text
   - Candidate answer
   - Expected rubric (Relevance, Specificity, Clarity, Ownership)
3. Gemini scores 0-6 per dimension
4. Score stored in assessment_responses
5. Final trust score = weighted average of all Q's
```

**Example Question & Scoring:**
```
Q: "Describe a situation where you debugged a complex backend issue. 
    What tools did you use, and how did you resolve it?"

Rubric:
- Relevance (0-6): Does answer directly address the question?
- Specificity (0-6): Are concrete details/tools mentioned?
- Clarity (0-6): Is explanation easy to follow?
- Ownership (0-6): Did candidate take responsibility?

Gemini Output:
{
  "relevance": 5,
  "specificity": 4,
  "clarity": 5,
  "ownership": 6,
  "avg_score": 5.0,  // 0-6 scale
  "reasoning": "Strong example, specific tools mentioned (Git bisect, 
                New Relic), clear methodology. Minor: could elaborate 
                on team communication."
}
```

**Final Trust Score Calculation:**
```
Component Scores (0-100):
- Psychometric: 72/100 (personality, communication)
- Behavioral: 68/100 (ownership, leadership)
- Skill:  75/100 (relevant experience, depth)
- Resume: 70/100 (validated against Q answers)

Trust Score = (Psychometric × 0.6) + (Behavioral × 0.4)
            = (72 × 0.6) + (68 × 0.4)
            = 43.2 + 27.2
            = 70.4 → **Final: 70/100**
```

### 4. Dynamic Question Generation (Gemini 1.5 Flash)

**Purpose:** Create unique questions per candidate assessment session (prevent cheating/memorization).

**Process:**
```
1. Assessment starts
2. For each category (Skill, Behavioral, Psychometric):
   - Retrieve 3-5 baseline questions from bank
   - Send to Gemini: "Rephrase these as stories/scenarios"
   - Gemini rewrites questions maintaining difficulty level
3. Questions served to candidate (different for each candidate)
4. Stored in assessment_responses for audit
```

**Example:**
```
Baseline Q: "Tell me about a leadership challenge you faced."

Gemini variations (for different candidates):
- "Walk me through a time your team disagreed with your direction."
- "Describe a moment where you had to make an unpopular decision."
- "Share an example where you developed someone on your team."

All test same competency (leadership) but phrased differently.
```

### 5. Market Intelligence & Career GPS (Gemini 1.5 Flash)

**Purpose:** Provide candidates with skill demand, competitive indices, salary trends.

**Process:**
```
1. Once candidate passes assessment
2. Backend queries: Skills of all verified candidates (in candidate pool)
3. For each skill, calculates:
   - Prevalence: % of candidates with skill
   - Demand: Jobs posted requiring this skill
   - Trend: Week-over-week change
4. Gemini generates insights: "Python is in high demand. You rank 
   in top 30% among candidates with Python + DataScience combo."
5. Cached (refreshed daily) to reduce API calls
```

### Key AI Guardrails

| Guardrail | Implementation |
|-----------|-----------------|
| **No Bias in Scoring** | Gemini prompt specifies criteria; ignores name/gender in answer |
| **Consistency Check** | If candidate answer contradicts resume, Gemini flags ("Resume says 5 yrs exp, but answer shows 2 yrs. Clarity: LOW") |
| **Hallucination Check** | If answer seems fabricated (too perfect, unverifiable), Relevance penalized |
| **Duplicate Q Prevention** | Questions regenerated each session; old questions not reused |
| **Prompt Injection Protection** | Candidate input sanitized; Gemini prompted to ignore instructions within answer |

## Security & Trust Model

### Authentication & Authorization

**Flow:**
```
1. Candidate/Recruiter signs up
2. Email verification (OTP)
3. Password creation (hashed via bcrypt)
4. JWT issued by Supabase Auth
5. JWT sent in Authorization header on every API call
6. Backend verifies JWT signature (Supabase public key)
7. User ID extracted from JWT claims
```

**JWT Payload:**
```json
{
  "iss": "https://xxx.supabase.co/auth/v1",
  "sub": "user-uuid",
  "email": "candidate@example.com",
  "email_verified": true,
  "user_metadata": {
    "role": "candidate"
  },
  "aud": "talentflow",
  "exp": 1711270000  // 1 hour expiry
}
```

**Protected Routes:**
- All `/dashboard/*` routes require JWT
- All `/api/*` endpoints verify user_id from JWT matches request payload
- Middleware redirects unauthenticated users to `/login`

### Anti-Cheat Mechanisms

**Tab-Switching Detection:**
```javascript
// Frontend
document.addEventListener('visibilitychange', () => {
  if (document.hidden) {
    tabSwitchCount += 1;
    if (tabSwitchCount === 1) {
      // Warning: "Stay on this tab or assessment will be disabled"
    }
    if (tabSwitchCount === 2) {
      // Disable question, mark as "Disqualified"
      // Reset password required to retry
    }
  }
});
```

**Copy-Paste Blocking:**
```javascript
document.addEventListener('copy', (e) => {
  e.preventDefault();
  showToast("Copying is disabled during assessment");
});
```

**Other Measures:**
- 60-second timer per question (auto-submit if timeout)
- Screen focus monitoring (optional camera proof)
- Permanent ban if caught cheating (account_status = Blocked)
- Cannot re-authenticate once blocked

### Data Privacy & Encryption

| Data | Encryption | Access |
|------|-----------|--------|
| **Passwords** | bcrypt (salt rounds: 10) | Never stored plain |
| **Resume PDFs** | AES-256 (Supabase Storage) | Only candidate + recruiters (RLS) |
| **Aadhaar/Passport** | AES-256 (Supabase Storage) | Only admin (audit trail) |
| **Messages** | TLS in transit, AES at rest | Thread participants only (RLS) |
| **Assessment Data** | Stored in PG, encrypted at rest | Candidate only (RLS) |

### Trust Scoring Model

**For Candidates:**
```
Trust Score Components:
1. Skill Validation (30%): Resume matches Q answers
2. Behavioral (30%): Ownership, integrity, communication
3. Psychometric (25%): Personality, problem-solving
4. Experience Consistency (15%): Role history matches claims

Scoring Logic: Weighted average (0-100 scale)
Min. score to pass: 40/100
Min. score for "Elite" badge: 80/100
```

**For Companies (CPS - Company Profile Score):**
```
CPS Components:
1. Strategic Intent (25%): Clear hiring vision
2. Candidate Value (25%): What do you offer?
3. Ethical Hiring (25%): Fairness commitment
4. Team Ownership (25%): Leadership accountability

Scoring Logic: Weighted average (0-100)
Min. score to hire: 50/100 (recruiters below 50 get limited features)
CPS Logic: MAX function (if re-assessed, only increase, never decrease)
```

### Trust-Based Feature Gating

| Feature | Gate |
|---------|------|
| **Post Job** | CPS ≥ 50 |
| **See Candidates** | CPS ≥ 50; assessment completed |
| **Schedule Interview** | Candidate ≥ 40, Company ≥ 50 |
| **Global Talent Pool** | Company CPS ≥ 70 (premium feature) |
| **Message Candidate** | Candidate must be shortlisted OR invited |
| **View Resume** | Application shortlisted OR interview scheduled |
