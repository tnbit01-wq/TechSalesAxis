## What Was Removed

❌ **Supabase Authentication Layer**
- `auth.users` table references (replaced with custom JWT + `users` table)
- Supabase JWT token validation
- Supabase Auth email OTP (replaced with AWS SES)

❌ **Supabase Real-time Features**
- `supabase_realtime` publications
- `auth.uid()` PostgreSQL functions
- Supabase Realtime client subscriptions

❌ **Supabase Row-Level Security (RLS)**
- RLS policies using `auth.uid()`
- Supabase-managed data isolation
- (Replaced with FastAPI middleware authorization)

❌ **Supabase Storage**
- Supabase Storage paths (kept only S3)
- Supabase signed URLs (kept only S3 signed URLs)

## What Replaced It

✅ **AWS RDS PostgreSQL**
- Region: ap-south-1 (Mumbai)
- Database: `techsalesaxis`
- Authentication: Password-based for direct access + Application-layer JWT
- Backups: Automated daily snapshots (7-day retention)

✅ **FastAPI JWT Authentication**
- Located in: `apps/api/src/core/auth.py`
- JWT issued by: Application backend
- Secrets stored in: AWS Secrets Manager
- User table: `public.users` (not `auth.users`)

✅ **AWS SES Email Service**
- OTP delivery: AWS SES
- Password resets: AWS SES
- Notifications: AWS SES
- Region: ap-south-1 (Mumbai)

✅ **AWS S3 Storage**
- Bucket name: `techsalesaxis-files`
- Region: ap-south-1 (Mumbai)
- Folder structure: `avatars/`, `resumes/`, `id-proofs/`, `company-logos/`, `company-assets/`, `posts/`, `uploads/`
- Signed URLs: Generated by FastAPI (S3 presigned URLs)

✅ **Real-time Features via WebSockets**
- FastAPI WebSocket endpoints
- Database polling for incremental updates
- Server-Sent Events (SSE) as fallback
- Located in: `apps/api/src/api/` (chat.py, notifications.py)
